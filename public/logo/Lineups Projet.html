Pour installer un nouveau projet Symfony, tu peux suivre ces étapes. Assure-toi d'avoir PHP, Composer, et Symfony CLI installés sur ta machine.

1. Installer Symfony CLI (si ce n'est pas déjà fait)

Le moyen le plus simple de créer un projet Symfony est d'utiliser la Symfony CLI.

Sur macOS/Linux

bash
curl -sS https://get.symfony.com/cli/installer | bash

Puis, ajoute le chemin d'accès au binaire dans ton ~/.bashrc, ~/.zshrc ou ~/.bash_profile:

bash
export PATH="$HOME/.symfony/bin:$PATH"

Sur Windows

Télécharge l'installateur depuis symfony.com/download.

2. Créer un nouveau projet Symfony

Utilise la Symfony CLI pour créer un nouveau projet. Ouvre ton terminal et exécute la commande suivante :

bash
symfony new mon_projet --webapp
  • mon_projet est le nom de ton projet.
  • L'option --webapp crée un projet avec les configurations et dépendances standard pour une application web.

3. Se rendre dans le répertoire du projet

Navigue vers le répertoire du projet créé :

bash
cd mon_projet

4. Lancer le serveur de développement

Pour lancer un serveur web local, utilise la commande suivante :

bash
symfony serve

Symfony CLI démarre un serveur local, et tu pourras accéder à ton application via http://localhost:8000.

5. Configurer ton environnement (Facultatif)

Par défaut, Symfony crée un environnement de développement. Tu peux configurer l'environnement de production ou de test en créant un fichier .env.local et en modifiant la variable APP_ENV.

6. Gérer les dépendances avec Composer

Symfony utilise Composer pour gérer les dépendances PHP. Si tu veux ajouter de nouvelles dépendances, utilise la commande composer require.

Exemple pour ajouter Twig

bash
composer require twig

7. Créer des bundles, contrôleurs, entités, etc.

Tu peux utiliser la Symfony CLI ou Composer pour générer des éléments spécifiques comme des contrôleurs, des entités, ou des formulaires.

Exemple pour créer un contrôleur

Qu'est-ce que CORS_ALLOW_ORIGIN ?

CORS_ALLOW_ORIGIN est souvent utilisé pour configurer la politique de partage des ressources entre origines multiples (CORS - Cross-Origin Resource Sharing). Cette variable définit les domaines autorisés à accéder à l'API depuis un autre domaine.

Résoudre le problème

Pour résoudre cette erreur, tu peux ajouter la variable CORS_ALLOW_ORIGIN dans ton fichier .env ou .env.local.

Exemple de Configuration dans .env

Ajoute cette ligne à ton fichier .env ou .env.local :

env
CORS_ALLOW_ORIGIN="*"

Cette configuration permet à tous les domaines d'accéder à ton API. Cela peut être utile pour le développement, mais ce n'est pas recommandé pour la production, car cela ouvre ton API à toutes les origines.

Limiter l'accès à certains domaines

Si tu souhaites restreindre l'accès à certains domaines spécifiques, définis-les ainsi :

env
CORS_ALLOW_ORIGIN="https://example.com, https://another-domain.com"

Cela autorisera uniquement les requêtes provenant de https://example.com et https://another-domain.com.

Configuration de CORS dans Symfony

Code Vue.js pour la connexion

Modifiez votre formulaire de connexion pour gérer l'envoi des données :

vue
<script setup>
import { ref } from "vue";
import axios from "axios";
import { useRouter } from "vue-router";

const email = ref("");
const password = ref("");
const errorMessage = ref("");
const router = useRouter();

const login = async () => {
  try {
    const response = await axios.post("http://127.0.0.1:8000/api/login_check", {
      username: email.value,
      password: password.value,
    });
    
    // Stocker le token JWT dans le localStorage
    localStorage.setItem("jwt", response.data.token);

    // Vérifier le rôle de l'utilisateur (cela dépend de votre API, mais souvent le rôle est codé dans le JWT)
    const tokenData = JSON.parse(atob(response.data.token.split('.')[1]));
    const roles = tokenData.roles || [];

    if (roles.includes("ROLE_ADMIN")) {
      // Redirection vers une route admin
      router.push("/admin");
    } else {
      // Gérer l'erreur si l'utilisateur n'est pas admin
      errorMessage.value = "Vous n'avez pas les droits pour accéder à cette page.";
    }
  } catch (error) {
    errorMessage.value = "Erreur lors de la connexion, veuillez vérifier vos identifiants.";
  }
};
</script>

3. Protéger les routes admin dans Vue Router

Modifiez votre routeur pour protéger les routes réservées aux administrateurs.

javascript
import { createRouter, createWebHistory } from "vue-router";
import Home from "../views/Home.vue";
import Admin from "../views/Admin.vue"; // Assurez-vous d'avoir une vue admin
import Connexion from "../views/Connexion.vue";

// Middleware de route pour vérifier les rôles
function adminGuard(to, from, next) {
  const token = localStorage.getItem("jwt");

  if (!token) {
    next('/connexion'); // Redirige vers la page de connexion si pas de token
  } else {
    const tokenData = JSON.parse(atob(token.split('.')[1]));
    const roles = tokenData.roles || [];

    if (roles.includes("ROLE_ADMIN")) {
      next(); // L'utilisateur est admin, autoriser l'accès
    } else {
      next('/connexion'); // Sinon, rediriger vers la connexion
    }
  }
}

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: "/",
      name: "home",
      component: Home,
    },
    {
      path: "/connexion",
      name: "connexion",
      component: Connexion,
    },
    {
      path: "/admin",
      name: "admin",
      component: Admin,
      beforeEnter: adminGuard, // Utiliser le guard pour protéger cette route
    },
  ],
});

export default router;

3. Créer un endpoint pour l'authentification

Dans src/Controller/SecurityController.php, créez une route qui permettra aux utilisateurs de se connecter :

php
// src/Controller/SecurityController.php
namespace App\Controller;

use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\JsonResponse;
use Symfony\Component\Security\Core\User\UserProviderInterface;
use Lexik\Bundle\JWTAuthenticationBundle\Services\JWTTokenManagerInterface;

class SecurityController
{
    public function login(Request $request, UserProviderInterface $userProvider, JWTTokenManagerInterface $JWTManager)
    {
        // Récupérez l'utilisateur à partir du username
        $user = $userProvider->loadUserByUsername($request->request->get('username'));

        // Créez le token JWT
        $token = $JWTManager->create($user);

        return new JsonResponse(['token' => $token]);
    }
}

3. Protéger les routes dans Vue Router

Configurez Vue Router pour protéger les routes qui nécessitent une authentification.

javascript
import { createRouter, createWebHistory } from 'vue-router';
import Home from '../views/Home.vue';
import Login from '../views/Login.vue';
import Secure from '../views/Secure.vue';

const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/login', name: 'Login', component: Login },
  { path: '/secure', name: 'Secure', component: Secure, meta: { requiresAuth: true } }
];

const router = createRouter({
  history: createWebHistory(),
  routes
});

router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !localStorage.getItem('auth_token')) {
    next({ name: 'Login' });
  } else {
    next();
  }
});

export default router;
